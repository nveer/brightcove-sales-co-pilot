What It Does Today

Four workflows, live right now

Every feature is triggered by a natural language prompt inside Claude Desktop. No dashboards to learn, no new apps to open.

📅
Daily Call Prep
Live

Every morning, a rep types "Prep me for my calls today." Co-Pilot pulls their full calendar, identifies every customer meeting, and builds a briefing card for each one — pulling in recent emails, account history, contract data, and suggested talking points.

Outcome: Reps walk into every call with full context, zero prep time.
📬
Inbox Triage
Live

Co-Pilot scans the inbox, separates signal from noise, and drafts responses to every email that needs one — pulling from Brightcove documentation to answer technical questions accurately. Reps review and approve before anything sends.

Outcome: Inbox handled in minutes. No response left behind.
🔍
Account Intelligence
Live

Ask "What's the status with Acme Corp?" and get a full account snapshot — contract value, product usage, renewal date, recent email threads, competitor context, and health assessment. All in under 30 seconds.

Outcome: Every rep can speak to any account like they've owned it for years.
📝
Post-Call Follow-Up
Live

After a call, Co-Pilot writes the follow-up email and creates a structured Notion page with action items, next steps, and key takeaways — automatically saved to the rep's account workspace. One prompt, one minute.

Outcome: Follow-ups sent faster, nothing falls through the cracks.

Before We Scale

Blockers to address

Co-Pilot is production-ready today for anyone using Claude Desktop with a standard account. Scaling it to the full enterprise team requires resolving a few IT and platform dependencies first.

⚙️ Claude Enterprise — IT Configuration Required
Requirement 1
Claude Desktop for All Reps
The plugin only works in Claude Desktop — not the web app
  • Each rep needs Claude Desktop installed on their Mac
  • Enterprise license must cover Claude Desktop (not web-only)
  • Cowork mode must be enabled for the organization
Requirement 2
MCP Connectors — Whitelisted
Admin Settings → Capabilities → Integrations
  • Gmail (read + write) — core inbox triage workflow
  • Google Calendar — required for daily call prep
  • Notion — required for follow-up pages and databases
  • Granola — optional, for live call companion
Requirement 3
External MCP Server — Whitelisted
Admin Settings → Capabilities → Network Access
  • Brightcove Data Gateway must be allowed as an external MCP connection
  • Endpoint: brightcove-data-mcps.public.bendingspoons.com
  • This is the source of all Salesforce, contract, and usage data
Requirement 4
Google & Notion OAuth Approval
Google Workspace Admin + Notion Admin
  • Google Workspace admin must approve Claude as an authorized OAuth app for Gmail and Calendar
  • Notion workspace admin must enable the Claude integration
  • One-time approval — reps then connect with one click
