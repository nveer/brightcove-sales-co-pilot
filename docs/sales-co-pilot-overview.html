What It Does Today

Four workflows, live right now

Every feature is triggered by a natural language prompt inside Claude Desktop. No dashboards to learn, no new apps to open.

📅
Daily Call Prep
Live

Pulls the full calendar, identifies every customer meeting, and builds a briefing card — email history, account data, and talking points included.

Outcome: Walk into every call with full context, zero prep time.
📬
Inbox Triage
Live

Scans the inbox, separates signal from noise, and drafts responses — pulling from Brightcove docs for technical accuracy. Rep reviews before anything sends.

Outcome: Inbox handled in minutes. No response left behind.

Live

Before We Scale

Blockers to address

Co-Pilot is production-ready today for anyone using Claude Desktop with a standard account. Scaling to the full enterprise team requires resolving a few IT and platform dependencies first.

⚙️ Claude Enterprise — IT Configuration Required
Requirement 1
Claude Desktop for All Reps
Each rep needs Claude Desktop installed with an Enterprise license that covers desktop access and Cowork mode enabled for the org.
Requirement 2
MCP Connectors — Whitelisted
The following MCP connectors need to be whitelisted in Claude Enterprise admin settings:
Gmail
Google Calendar
Notion
Granola
Requirement 3
External MCP Server — Whitelisted
Admin Settings → Capabilities → Network Access
The Brightcove Data Gateway must be approved as an external MCP connection — this is the source of all Salesforce, contract, and usage data.
brightcove-data-mcps.public.bendingspoons.com
Requirement 4
Google & Notion OAuth Approval
Google Workspace Admin + Notion Admin
Google and Notion must be approved within the Claude Enterprise environment so all employees can connect with one click.
🔮 Future Capabilities — Platform Dependencies

Gong Intelligence

🔒 What's needed: Gong call transcripts exported into the Brightcove Gateway MCP. Once available, this ships in days — the Co-Pilot integration is already built and waiting.
