Integrations

Platforms It Connects To

Each platform requires its own setup. Click any row to see connection instructions.

🎤
Gong
Call recordings · Transcripts · SE team call history
API Key Needed

Used for pulling call transcripts, searching SE call history, and surfacing best practices. You'll need a Gong API Access Key + Secret from your org's Gong admin.

1
Go to Gong → Settings → API (or ask your Gong admin to create API credentials for you)
2
Create a new API Key. Copy the Access Key and Access Secret.
3
Find your org's Base URL — it looks like https://us-XXXXX.api.gong.io/v2. It's shown in the Gong API settings page.
4
Open scripts/.env in your plugin folder and fill in:
# Gong API credentials GONG_ACCESS_KEY=your_access_key_here GONG_ACCESS_KEY_SECRET=your_access_secret_here GONG_BASE_URL=https://us-XXXXX.api.gong.io/v2
5
Open skills/gong/gong_api.sh and update the SE_USER_IDS array at the top with your team's Gong user IDs. Ask your Gong admin or pull them from the Gong API.
🎟️ Need API access? Gong API keys require admin setup. Submit a ticket to IT/Support and ask them to create Gong API credentials for you. Include your name and team in the request.
⚠️ Gong API keys are org-level — they give access to your org's Gong data. Do not share your credentials with someone outside your org. Each team member should create their own API key.
☁️
Salesforce
Account data · Opportunities · Product info · Contract details
API Key Needed

Used to pull account details, active opportunities, and contract data for call prep and account summaries. Uses Salesforce's OAuth 2.0 Password Flow — you'll need a Connected App from your Salesforce admin.

1
Ask your Salesforce admin to create a Connected App for you in Setup → App Manager. Enable OAuth and request scopes: api, refresh_token.
2
After the app is created, copy the Consumer Key and Consumer Secret.
3
Get your Salesforce Security Token: in Salesforce go to My Settings → Personal → Reset My Security Token. It'll be emailed to you.
4
Open scripts/.env and fill in your Salesforce section:
# Salesforce credentials (OAuth 2.0 Password Flow) SF_CONSUMER_KEY=your_consumer_key SF_CONSUMER_SECRET=your_consumer_secret SF_USERNAME=your.email@company.com SF_PASSWORD=your_salesforce_password SF_SECURITY_TOKEN=your_security_token SF_TOKEN_URL=https://login.salesforce.com/services/oauth2/token
🎟️ Need a Connected App? Salesforce Connected Apps require admin setup. Submit a ticket to IT/Support asking them to create a Salesforce Connected App with OAuth password flow enabled. Include your Salesforce username in the request.
🔒 Critical: These are your personal Salesforce login credentials. Every query runs as you and appears in Salesforce audit logs under your name. Never share your credentials — each user must configure their own.
📧
Gmail
Email read/search/draft · Inbox triage · Thread context
MCP Connector

Used for reading email threads, drafting responses, and archiving noise during email triage. Connects via two Claude Desktop MCP connectors (read + write).

1
In Claude Desktop → Settings → Integrations, search for "Gmail". You'll connect your own Gmail account — each person must connect their own.
2
Connect both the read connector and the write connector to your Google account.
3
Authorize Google's OAuth consent screen — grant the requested Gmail permissions. You're authorizing access to your personal/work Gmail inbox.
4
Once connected, restart Claude Desktop and run /prime to verify Gmail is accessible.
ℹ️ The email triage command never auto-sends emails. It always shows you draft content and asks for confirmation before sending anything.
📆
Google Calendar
Schedule · Meeting classification · Daily timeline
MCP Connector

Used in daily call prep to pull your schedule, classify meetings as customer vs internal, and generate the day's timeline. Read-only — this plugin cannot create or modify calendar events.

1
In Claude Desktop → Settings → Integrations, search for "Google Calendar". Connect your own Google Calendar account via OAuth.
2
Grant read-only access to your calendar. This is personal — it only accesses the calendar of whoever connects it.
3
Once connected, restart Claude Desktop to complete the setup.
✅ Read-only access only. This plugin cannot create, modify, or delete any calendar events.

Notion
Call follow-up pages · Account tracking · Knowledge base
MCP Connector

After each call, SE Command Center writes a structured follow-up page to Notion with action items, resources, and a copy-paste follow-up email. You'll need to set up your own Notion databases and update the IDs in your config.

1
Connect Notion in Claude Desktop → Settings → Integrations
2
In your Notion workspace, create two databases:
• Customer Call Prep (properties: Name, Customer, Date, Status, Attendees)
• Call Follow-Ups (same properties)
3
Copy each database's ID from the URL (the UUID after the last /)
4
Open CLAUDE.md and update the Notion Databases section with your database IDs and parent page ID
## Notion Databases — update with YOUR IDs Customer Call Prep DB: collection://your-collection-id Call Follow-Ups DB: collection://your-collection-id Your parent page: your-page-id

Granola
Meeting transcripts · Live call notes · AI summaries
MCP Connector

Used by the live Call Companion to monitor meeting notes in real time and pull transcripts post-call. Requires the Granola desktop app and its MCP connector.

1
Install Granola on your Mac (it records and transcribes meetings in the background)
2
In Granola settings, enable the MCP / Claude integration
3
Connect Granola in Claude Desktop → Settings → Integrations
4
Granola will automatically make your meeting transcripts accessible to Claude
🙌 No IT ticket needed! Granola is fully self-serve. Once installed, it automatically records and transcribes all your meetings. You can search across ALL your recorded conversations directly from the Granola app — no admin involvement required. This makes it the fastest integration to get up and running.
Installation

First-Time Setup

Get from zero to running in about 30 minutes. Most of the time is waiting for Salesforce admin to set up your Connected App.

1
Install the plugin in Claude Desktop. Double-click the .plugin file or drag it into Claude Desktop's plugin panel.
2
Select a workspace folder in Claude Desktop (Cowork mode). This is where all your outputs, call prep pages, and context files will live.
3
Create your scripts/.env file by copying scripts/.env.example and filling in your credentials (Gong + Salesforce).
4
Connect MCP integrations in Claude Desktop: Gmail (read + write), Google Calendar, Google Drive, Notion, and optionally Granola. See the Platforms section above for instructions on each.
5
Update CLAUDE.md with your details: your name, role focus, Notion database IDs, and any account-specific context you want Claude to know.
6
Update Gong SE user IDs in skills/gong/gong_api.sh — replace the default IDs with your team's actual Gong user IDs. Your Gong admin can supply these.
7
Run /prime in Claude Desktop to verify everything is connected and Claude can read your workspace context.
Distribution

Hosting on GitHub (For Plugin Maintainers)

Host the onboarding page on GitHub Pages and the plugin file on GitHub Releases — then share a single URL with your whole team.

1
Create a private GitHub repo named brightcove-se-command-center. Push all plugin files to it.
2
Enable GitHub Pages: Go to repo Settings → Pages → Source: Deploy from branch → select main → folder /docs. Move se-plugin-onboarding.html to a docs/ folder and rename it index.html.
3
Create a GitHub Release: Go to Releases → Draft a new release → tag it v1.0.0 → upload se-command-center.plugin as a release asset → publish.
4
Update the Download button URL in docs/index.html: replace ~~your-github-username~~ with your actual GitHub username. The final URL will be:

https://github.com/YOUR-USERNAME/brightcove-se-command-center/releases/latest/download/se-command-center.plugin
5
Share this single URL with your SE team:

https://YOUR-USERNAME.github.io/brightcove-se-command-center/

They click Download Plugin → double-click the file → Claude Desktop installs it automatically.
🔒 Make the repo private — it contains Brightcove-specific workflow files. GitHub Pages works on private repos with a GitHub Teams or Enterprise plan. Alternatively, use a public repo but ensure no sensitive info is in any committed file (all credentials stay in the local .env which is gitignored).
Security

Why You Need Your Own API Tokens

Sharing API credentials is not just a security risk — it breaks the plugin for everyone involved. Here's why.

❌ If You Use Someone Else's Token
All API calls appear as the token owner in audit logs. In Salesforce, you'll query their data under their access permissions — and they can see everything you searched. In Gong, their SE team filters apply, not yours.
✅ With Your Own Token
Queries run as you — your data, your access permissions, your audit trail. Gong SE filters point to your team's calls. Salesforce shows your accounts. Everything is scoped correctly.
⚠️
Salesforce is especially sensitive. The plugin uses an OAuth password flow that includes your actual username and password. Sharing this gives someone full access to your Salesforce instance with your permissions. Always use your own credentials.
ℹ️
Gong Base URL is org-specific. The URL (https://us-XXXXX.api.gong.io) identifies your Brightcove Gong instance. Everyone at Brightcove uses the same base URL, but needs their own API key to authenticate against it.

Available Commands

Type any of these in Claude Desktop after installing the plugin.

/prime
Run at the start of every session. Reads all context files and confirms Claude is up to speed.
/daily_prep
Generates a dark-theme HTML call prep page for today. Pulls calendar, enriches customer meetings with Gong + Gmail context.
/call_companion
Live call assistant. Monitors Granola notes, researches docs in real time, then writes a Notion follow-up page after the call.
/email_triage
Scans inbox, categorizes by action, drafts responses, and archives noise. Always reviews with you before sending anything.
/call_prep [customer]
On-demand pre-call briefing for any account. Pulls Gong history, Salesforce data, and recent emails.

Post-call capture. Records outcomes, action items, and customer sentiment. Updates context files.

360° account view: 6 months of call history, full Salesforce pull, email threads, and health score.
