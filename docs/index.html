Overview

What This Plugin Does

Brightcove Sales Co-Pilot plugs into Claude Desktop and gives you a set of commands that automate the most time-consuming parts of your sales workflow.

🌅
Daily Call Prep
Say "prep me for my calls today" and Claude pulls your full day together — Google Calendar, Gong call history, Granola meeting notes, and Gmail — then outputs a visual HTML briefing page. Any open customer question is auto-researched on support.brightcove.com and embedded with a ✅ before your first call.
Morning Ritual
📞
Live Call Companion
Monitors your Granola transcript in real time during a call. When it detects a trigger phrase, it checks Gong Sales team call history first to see if a colleague has solved this before, cross-references Google Docs, then deploys parallel agents to research answers — all while you keep talking. After the call: one Notion page + one copy-paste follow-up email, already written.
Real-Time
📬
Email Triage
Reads full email threads before drafting — never contradicts what you already sent. Auto-researches answers on support.brightcove.com and embeds them in the draft. Won't archive support emails if you're named, the customer is frustrated, or the thread is stalled. Nothing sends without your approval.
Inbox Zero
📊
Account Summary
Full 360° account view on demand — pulls Gong history, Gmail threads, Brightcove Gateway data, and Notion notes simultaneously. Surfaces cadence gaps, competitor risk, renewal timing, and upsell signals. One brief ready to share with your Account Executive or bring into an exec conversation.
Account Intel
⚔️
Competitor Analysis
Mines your Gong call history for how Brightcove has won (or lost) against a competitor before, then cross-references Brightcove support docs for feature differentiators. Output: a structured battlecard with their strengths, your advantages, and objection responses — built from your own calls.
Competitive
🗂️
Notion Integration
After every call, automatically writes a structured Notion follow-up page: action items, resources, key intel, and one copy-paste follow-up email at the bottom covering all topics. Consistent format — [Customer] — [Date] — enriched with Gong and Brightcove Gateway data. No manual note-taking required.
Auto Follow-Up
🧠
Always-On Research Engine
Built into every command. Any unanswered technical question in a draft, prep card, triage summary, or follow-up page triggers an automatic lookup at support.brightcove.com. Answers appear inline with a ✅ Researched Answer marker. You never walk into a call with an open question still unanswered.

Integrations

Platforms It Connects To

Most integrations are self-serve with no IT tickets required. Click any row for connection instructions.

🔵
Brightcove Gateway
Account data · Opportunities · Usage metrics · Contract details · Salesforce exports
Auto-Configured ✅

Used to pull account details, active opportunities, usage data, and contract information via Brightcove's internal BigQuery data warehouse. No OAuth or Salesforce Connected App required — connects via MCP connector and uses your Brightcove credentials.

1
In Claude Desktop → Settings → Integrations, search for "Brightcove Gateway" and install the MCP connector.
2
Authenticate with your Brightcove corporate credentials when prompted — this connects to the brightcove-lumenx-42 BigQuery project.
3
Once connected, restart Claude Desktop. The plugin will automatically use the Gateway for all account and usage queries.
✅ No IT ticket needed! The Brightcove Gateway MCP connector is self-serve. Authenticate with your existing Brightcove credentials — no Connected Apps, no OAuth setup, no admin involvement required.
ℹ️ The Gateway connects to BigQuery project brightcove-lumenx-42, dataset external_shared_views. This includes enriched Salesforce exports, contract data, entitlement usage, and Video Cloud event metrics — all the data you'd otherwise need Salesforce access to pull manually.

Gmail
Email read/search/draft · Inbox triage · Thread context
MCP Connector

Used for reading email threads, drafting responses, and archiving noise during email triage. Connects via two Claude Desktop MCP connectors (read + write).

1
In Claude Desktop → Settings → Integrations, search for "Gmail". You'll connect your own Gmail account — each person must connect their own.
2
Connect both the read connector and the write connector to your Google account.
3
Authorize Google's OAuth consent screen — grant the requested Gmail permissions. You're authorizing access to your personal/work Gmail inbox.
4
Once connected, restart Claude Desktop and run /prime to verify Gmail is accessible.
ℹ️ The email triage command never auto-sends emails. It always shows you draft content and asks for confirmation before sending anything.
📆
Google Calendar
Schedule · Meeting classification · Daily timeline
MCP Connector

Used in daily call prep to pull your schedule, classify meetings as customer vs internal, and generate the day's timeline. Read-only — this plugin cannot create or modify calendar events.

1
In Claude Desktop → Settings → Integrations, search for "Google Calendar". Connect your own Google Calendar account via OAuth.
2
Grant read-only access to your calendar. This is personal — it only accesses the calendar of whoever connects it.
3
Once connected, restart Claude Desktop to complete the setup.
✅ Read-only access only. This plugin cannot create, modify, or delete any calendar events.
🗂️
Google Drive
File access · Document context · Reference materials
MCP Connector

Used for accessing reference documents, customer files, and any Google Drive content you want Claude to have context from.

1
In Claude Desktop → Settings → Integrations, search for "Google Drive"
2
Connect your own Google Drive account via Google OAuth
3
Authorize the requested permissions — Claude can only read files you explicitly share with it
4
Once connected, you can reference Google Drive documents in commands like /call_prep or /email_triage
✅ Read-only access. Claude can search and read files you've shared, but cannot create, modify, or delete Google Drive files.

Notion
Call follow-up pages · Account tracking · Knowledge base
Auto-Configured ✅

After each call, the plugin writes a structured follow-up page to Notion — action items, resources, and a copy-paste follow-up email. No manual database setup needed — onboarding creates your Call Follow-Ups and Customer Call Prep databases automatically.

1
Connect Notion in Claude Desktop → Settings → Integrations (sign in with your Brightcove Notion account)
2
That's it. During onboarding, Claude automatically creates your Call Follow-Ups and Customer Call Prep databases in Notion — no IDs to copy, no manual config.
✅ Zero manual setup! Notion databases are created automatically during first-run onboarding. You never need to touch database IDs, CLAUDE.md, or any config files.

Granola
Meeting transcripts · Live call notes · AI summaries
MCP Connector

Used by the live Call Companion to monitor meeting notes in real time and pull transcripts post-call. Requires the Granola desktop app and its MCP connector.

1
Install Granola on your Mac (it records and transcribes meetings in the background)
2
In Granola settings, enable the MCP / Claude integration
3
Connect Granola in Claude Desktop → Settings → Integrations
4
Granola will automatically make your meeting transcripts accessible to Claude
🙌 No IT ticket needed! Granola is fully self-serve. Once installed, it automatically records and transcribes all your meetings. You can search across ALL your recorded conversations directly from the Granola app — no admin involvement required. This makes it the fastest integration to get up and running.
Installation

First-Time Setup

Get from zero to running in about 10 minutes. Most integrations are self-serve — no IT tickets required.

1
Download the plugin from the button at the top of this page or in the hero section.
2
Open the file — double-click the .plugin file. Claude Desktop will open and show a Save Plugin confirmation dialog.
3
Click Save in the Save Plugin dialog. This plugin is verified as a Safe Plugin — it installs cleanly with no terminal commands or admin access required.
💡 If Claude asks permission to take action — click Always Allow. This lets the plugin run workflows on your behalf without asking every time.
4
Connect your integrations — Claude will walk you through connecting Gmail, Google Calendar, and Notion via Claude Desktop → Settings → Integrations. Each takes about 30 seconds.
5
Run onboarding — Claude will greet you, collect your name and role, and automatically set up your Notion workspace. Done in ~5 minutes.
Reference

Available Commands

Type any of these in Claude Desktop after installing the plugin.

/prime
Run at the start of every session. Reads all context files and confirms Claude is up to speed.
/morning_schedule
Daily 7am briefing. Loads Google Calendar, classifies customer vs internal meetings, prints today's schedule.
/daily_prep
Generates a dark-theme HTML call prep page for today. Pulls calendar, enriches customer meetings with Gong + Gmail context.
/call_companion
Live call assistant. Monitors Granola notes, researches docs in real time, then writes a Notion follow-up page after the call.
/email_triage
Scans inbox, categorizes by action, drafts responses, and archives noise. Always reviews with you before sending anything.
/call_prep [customer]
On-demand pre-call briefing for any account. Pulls Gong history, Brightcove Gateway data, and recent emails.
/account_summary [customer]
360° account view: 6 months of call history, full Brightcove Gateway pull, email threads, and health score.
